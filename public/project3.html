ISD Course Design Process Application

Explore the comprehensive case study of the ISD Course Design Process App, a digital solution developed by KeelWorks to streamline educational and corporate training course design.

Project Overview

This project addresses the inefficiencies of decentralized documentation and redundant data entry in the course design process by introducing a unified, role-based application.

Why Build It?

The ISD Course Design Process App consolidates documentation and workflows to enhance efficiency, improve version control, and streamline stakeholder approvals.

Application Aim

The app aims to unify documentation, streamline the approval process, enhance collaboration, and support efficient course updates.

Key User Roles and Permissions

  • Instructional System Designer (ISD): Full creation and editing capabilities.
  • Stakeholder: Approval and feedback provision.
  • Subject Matter Expert (SME) & Employees: Limited viewing access.
  • Support Staff: Design and technical support with restricted editing.

Challenges Addressed

The application tackles inefficiencies in documentation, issues with version control, and collaboration hurdles within the course design process.

Step-by-Step Process Overview

The ISD Course Design Process App features a structured workflow for different user roles, from course creation by ISDs to approval by stakeholders and viewing by SMEs and employees.

Application Interface and Navigation

The interface features a horizontal tab layout that displays various stages of the course design process, with role-based access control enhancing security and user experience.

Conclusion

This case study highlights how the ISD Course Design Process App streamlines and centralizes the course design process at KeelWorks, enhancing productivity and potentially serving as a commercial product for other organizations.
